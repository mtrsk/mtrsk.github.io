Split Brain September 28, 2024 by Marcos Benevides

:ID: 582a4090-eb66-4dfd-8166-9acd3c97dcc9

Split brain means that the [BROKEN LINK: d8a1a1ff-47e6-44bc-a627-83ca8dc61ecb] is split in two (or more) parts, but both parts think they are the only remaining part of the cluster. This can lead to very bad situations when both parts of the cluster try to host the resources that are offered by the cluster. If the resource is a file system, and multiple nodes try to write to the file system simultaneously and without coordination, it may lead to corruption of the file system and the loss of data. As it is the purpose of a high availability cluster to avoid situations where data could be lost, this must be prevented no matter what. (, Van Vugt 2014, 6)

References

Van Vugt, Sander. 2014. Pro Linux High Availability Clustering. Apress.
